# 端到端示例（原型 J）：机器人末端硬件改装资料（机械 + 电气规格书）

演示 **原型 J·硬件末端/物理接口规格与改装资料类** 怎么跑一遍：核心不是"底层支不支持"（硬件已存在），而是**给客户开放一份精确到能据以加工、接线、验收的物理接口规格书**——错一个数字（承重/力矩/供电/pinout）就可能让客户把自制配件接到会动、会伤人的机器人上。**这是一份需求文档**，规定"这套对外资料必须包含什么、精确到什么程度、有哪些安全红线"，**不是硬件数据本身**——所有真实数值都写成占位 `[待硬件/机械/电子团队填]`，由内容真相源团队填。示例中的取值仅为占位演示。

## 输入

> 有客户想在我们机器人末端装自己的配件（换个夹爪 / 加个相机支架之类），问我们能不能给末端的机械装配和电气接线资料，让他们自己做配件接上去。

- 关联特性 ①『官网"硬件改装"板块』= developer.galbot.com 文档 IA + 角色分级门控，owner=DevRel/站点前端（独立成文，互指）。

- **前置元决策**：每型号参数不同 → **每型号独立立项、各出一份规格书**（本档以 **G1** 为代表；G3 及其它型号照此模板各自立项；型号清单本身 `[待确认]`）。

**第 3 步归类** → **L-Tier = N/A**（对外交付/资料形态，非运行时能力、不暴露软件 API、不占 API 预算）。重心=内容真相源团队（硬件/机械/电子）+ 文档门户层。

**第 4 步跨层** → 触底=**内容由硬件/机械/电子团队定义**（定义权在工程图纸/PCB/BOM）。owner=**硬件/机械/电子团队（真相源）+ 文档门户层（承载）**，⚠ **别默认派二开/文档团队照二手资料写**（照二手资料写=图纸-实物不一致的返工与事故源头）。命中**外部依赖轴**的硬件版：**型号 × 硬件版本/批次**（同型号改板会改孔位/pinout）。

**原型 J 六问 + 复用 H（要点）**：
- ① 物理契约矩阵？→ 机械（法兰/孔位阵列+螺纹/定位销/尺寸公差带/静·动态承重/各轴力矩上限/质心惯量/安装姿态）+ 电气（连接器料号+配对件/逐针脚 pinout/线序/供电电压·电流·功率含峰值/通信协议/接地屏蔽）**逐格给死、可据以加工与接线验收**；本档所有数值=占位待填。
- ② 型号 × 硬件版本矩阵？→ 每型号独立立项（本档=**G1**）；规格书标**适用硬件版本范围/序列号区间/固件依赖**，锁到批次。
- ③ 图纸-实物一致性防腐（复用 H④，对象换成工程图纸）？→ 每个数据源头标明来自哪份**工程图纸/PCB/BOM 版本**、谁**背书与实物一致**、图纸改版时规格书**同步机制**。
- ④ 物理安全红线（强命中，务必问一次）？→ 承重/力矩/供电红线**显著警示 + 超限后果**；**禁改区**（动了会解除急停/安全机制的接口）；改末端后需**负载重标定/安全参数申报**；改装授权/资质、保修与责任边界、免责与合规声明。
- ⑤ 交付物精度与自验？→ 可下载 **STEP/DXF/CAD 图纸 + 3D 模型 + PDF**；**公制 + ISO 公差标注 + 坐标系/基准面**；给客户**关键配合尺寸检验清单 / 对配样件**自验"我加工的配件能装上"。
- ⑥ 可仿照性（复用 H⑥，对象=机械/电气设计）？→ 关联特性 ②给一个**参考末端的完整机械+电气设计 + 换件适配步骤**供照抄改（独立成文）。
- 开放范围（复用 H①责任边界）？→ **仅对已购机客户开放**，在 developer.galbot.com 做**用户角色分级门控**（未购机/未授权看不到详细图纸/pinout）；官方正式支持、随硬件版本更新、版本化 `[待确认·产品拍板]`。

## 物理安全红线（强命中，显著警示）
> ⚠ **给错一个数字 = 客户把自制配件接到会动、会伤人的机器人上，出人身安全事故 + 法律责任。** 以下必须在资料中**显著、醒目**呈现（不是埋在正文里）。
- **承重 / 力矩 / 供电红线** + **超限后果**：超承重/力矩 → 触发保护/机械损坏/失控；超供电包络 → 器件损坏/起火。逐项写清超限会发生什么。
- **禁改区**：`[待硬件填]`——哪些接口**绝不可改动**，动了会**解除急停/安全围栏/碰撞检测**。改这些直接列为禁止。
- **改末端后的合规动作**：装了新配件（改变负载/质心）后须做**负载重标定 + 安全参数申报** `[流程待填]`，否则安全监测失效。
- **改装授权/资质、保修与责任边界**：谁有资格改装 `[待产品/法务填]`；改装是否**失保**、故障责任归属 `[待填]`；**免责与合规声明**（本资料仅供合规改装，超红线使用后果自负）`[待法务填]`。
- **开放范围门控**：详细图纸/pinout **仅对已购机+授权客户可见**（角色门控），未授权看不到——避免物理接口数据外泄被误用。

## 图纸-实物一致性与防腐
> 硬件类最大长期返工源：图纸改了、对外规格书没跟 = 客户按过时数据加工的配件装不上、或按旧 pinout 接线烧板。
- **每个数据源头**：契约表每格都标来自哪份**工程图纸/PCB/原理图/BOM 的编号与版本**（见上表"来源/背书"列）。
- **背书人**：机械值由硬件/机械团队指定人背书"与实物一致"，电气值由电子团队背书；未背书的格子不得对外发布。
- **同步机制**：图纸/PCB 改版 → 触发规格书**版本化更新** `[机制待定：随硬件版本发布 / 评审签署]`；老硬件版本的旧规格书保留并标注适用序列号区间，不覆盖。

## 交付物精度与客户自验
- 交付形态：网页可读 + **可下载 STEP / DXF / CAD 图纸 + 3D 模型 + PDF**（在 developer.galbot.com，角色门控后可下）。
- 标注标准：公制 mm、ISO 公差、坐标系与基准面统一声明（见对外物理契约"通用标注前提"）。
- **客户自验清单**：给一份**关键配合尺寸检验清单**（法兰对接面平面度、孔位坐标、定位销孔、连接器机械位置等），客户加工后据此自检"能不能装上"；可选提供**对配样件**做实物对配。

## 测试 / QA
> 测试对象是**资料本身的完整性、精度、可据以加工验收、以及安全与门控是否到位**——不是软件功能。
- **契约完整性**：机械表 + 电气表**每一格都有真实值**（无遗留占位）、每格都有图纸/PCB 来源与背书人；单位制/公差标准/坐标系基准齐全。
- **可加工验收**：由中立方按契约表 + CAD 图纸加工一件样件，验证能装上（落公差带内）+ 能接通（pinout 对）——**外部工程师只靠这份资料能做出装得上的配件**即达标。
- **安全红线**：承重/力矩/供电红线 + 超限后果 + 禁改区 + 重标定/申报要求 + 免责/授权，**逐条在资料中显著可见**。
- **图纸-实物一致**：抽查若干格，核对对外值与最新工程图纸/PCB 一致、背书人签署在案。
- **门控生效**：未授权角色访问 developer.galbot.com 该资料，**看不到**详细图纸/pinout（验证角色分级门控）。
- **矩阵覆盖**：本档只覆盖 **G1 × 适用硬件版本/序列号区间**；其它型号/批次不在本档验收范围。

## 依赖与前置条件
- 依赖硬件/机械/电子团队从工程图纸/PCB/BOM 提供并背书真实数值（本特性只规定"要哪些、精确到什么程度"，不产生数据）。
- 依赖 developer.galbot.com 具备**用户角色分级门控**能力（属关联特性 ①/门户层）。
- 前置：G1 硬件版本/序列号区间与固件依赖已确定。
