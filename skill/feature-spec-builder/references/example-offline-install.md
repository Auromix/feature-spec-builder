# 端到端示例（原型 I）：SDK 离线/本地安装

演示 **原型 I·打包/分发/离线安装/部署类** 特性怎么跑：核心不是"底层支不支持"（能力已存在），而是**依赖捆绑、平台矩阵、完整性验签、安装态生命周期、权限落点、离线失败可诊断**。示例中的取值仅为占位演示。

---

## 输入

> 客户有时通过网络下载安装，网络不好会失败；或内网/气隙环境不能联网。需要能本地离线安装，且**足够简单**：拿到包、看 README、一行命令 + 少量交互就装好。

## 澄清过程（要点，一次一问）

**第 1 步复述** → 给已有联网安装的 SDK（C++ core + pybind11）补一条**离线安装路径**，一行命令 + 少量交互装好并自检，补充而非替换联网安装。对吗？→ 对。

**第 2 步分流** → 主特性=『离线安装包 + 一行命令安装器』（捆绑/矩阵/形态/传输/校验幂等/权限/ABI/签名全是内部维度，一份 md）；关联特性=『离线 README』（共享真相源，同 md 内成节）、『装后自检 verify』（作安装收尾步）。

**第 3 步归类** → **L-Tier = N/A**（交付/分发形态，非运行时能力、不占 API 预算）；重心落**构建/分发基建层**。

**第 4 步跨层** → 触底 **A 纯二开产物可交付**（不下探固件/算法/硬件）。owner=**构建/分发基建层（SDK 构建/DevOps）+ 文档门户层**，⚠ **不派二开组**。强命中**外部依赖轴**：glibc/ABI × CPython × OS 发行版矩阵与 EOL。

**原型 I 五问 + 复用 H（要点）**：
- ① 依赖捆绑？→ **完全自包含**（bundle 系统库/Python wheels，落地即装）。⚠ **再分发许可前置 gate**：若涉 CUDA redistributable / GPL-LGPL 静态链接，先核"哪些能合法 bundle、哪些客户自带" `[待确认·法务核]`。
- ② 安装形态 + 静默边界？→ **自解压 `install.sh`**（tarball + 脚本）；一行 `./install.sh`；交互=选安装路径 / 接受许可 / 验签确认；**提供 `--yes` + 应答文件**供气隙无人值守批量部署（验签不静默跳过）。
- ③ 完整性？→ **sha256 清单 + GPG 签名 + 公钥带外分发 + README 强制验签**，**验签失败 fail-closed 硬阻断**。bundle 内第三方 license/SBOM 随包附 `[待确认·合规]`。
- ④ 安装态生命周期？→ 装后**自检**（import + 版本 + `.so`/ABI 探测，**纯软件、不依赖真实硬件**）；**幂等重入**；装进**隔离 venv**、与既有安装/旧版本**并存不冲突**；MVP 含幂等，**原地升级/卸载分期** `[MVP 边界]`。
- ⑤ 权限落点？→ **默认纯用户级**（装进用户目录/venv，**免 root**、不污染系统 Python）；系统级为可选 + 二次确认。
- 平台矩阵（复用 H②+外部依赖轴）？→ MVP 单格 **x86_64 × Ubuntu 22.04 × Py3.10 × glibc2.35**；arm64/多 Python 分期；`.so` 优先 **abi3** 缩矩阵，否则 per-version。
- 支持等级（复用 H①）？→ 安装是核心路径，倾向**官方支持**（随 SDK 每版发离线包、双路径 CI 防漂移）`[待确认·拍板]`。

## 一句话定义
提供 SDK 的**完全自包含离线安装包**：客户在断网机器上拿到包、照 README 跑 `./install.sh`（+ 选路径/验签少量交互），即装进用户级隔离环境并自检通过。补充既有联网安装，不替换。

## 能力定位
- 层级：**N/A**（交付/分发形态，非运行时能力、不暴露 API、不占 API 预算）
- 实现层：构建/分发基建层（主，owner=SDK 构建/DevOps [待确认·指派]）+ 文档门户层（README）；触底=A 纯二开产物，不下探固件/算法/硬件。
- 外部依赖轴：glibc/ABI × CPython × OS 发行版矩阵与 EOL（不归内部部门）。

## 范围边界
- In：完全自包含离线包（bundle 系统库+Python wheels）；`install.sh` 一行命令 + 少量交互 + `--yes` 静默模式；sha256+GPG 验签(fail-closed)；装后自检(纯软件)；幂等；用户级隔离 venv 安装；README(含验签/前置矩阵/失败排查)。平台 MVP：x86_64/Ubuntu22.04/Py3.10。
- Out：原地升级、干净卸载/回滚（分期，Out 显式声明）；arm64/多 distro/多 Python（分期）；系统级安装为可选非默认；不可合法 bundle 的依赖（客户自带）。
- 关联特性：README（同 md 成节，真相源=安装器行为）；未来升级/卸载能力。

## 开发者/集成商故事
- **[P1]** 作为客户现场工程师，我希望在**断网**目标机上拿离线包、照 README 跑一行命令+少量交互，就装好并看到自检通过。
  - 验收（DX 硬锚点，仿 H⑥换到安装场景）：**一名无该 SDK 背景的工程师，在断网的 x86_64/Ubuntu22.04 机器上，只靠 README，≤15 分钟内离线装通并通过自检、全程不求助**。
  - 验收（工程）：验签失败 fail-closed 阻断；重复跑幂等；免 root 装进用户 venv；失败给可读根因+建议动作。
  - 优先级依据：客户硬需求（网络差/气隙无法联网安装）。
- **[P2]** `--yes`/应答文件无人值守批量部署。
### MVP：单平台 full-bundle + 一行命令装通 + 验签 + 自检 + 幂等；升级/卸载/多平台分期。

## 行为规格
- 正常流程：校验 sha256+GPG（失败即停）→ 交互选路径/接受许可 → 建隔离 venv → 释放 bundle 装入 → 装后自检 → 打印结果+退出码。
- 幂等：重复跑对同目标安全（检测已装→跳过/覆盖同版）。
- 失败处理（离线唯一 UX）：每类失败（缺库/Python 版本不符/ABI 不匹配/验签失败/权限不足）给**人话根因+建议**，落可回传诊断包（环境探测+校验结果+分级退出码）。
- 权限：默认用户级免 root；系统级需 sudo + 二次确认。
- 可观测：自检输出清单 + 退出码分级；诊断包（不出域，客户自留）。

## 非功能性需求(NFR)
- **支持等级**：官方支持[待确认]（随版本发离线包、双路径 CI 回归）。
- **安全（红线，强命中）**：GPG 验签 fail-closed；公钥带外可信分发；默认免 root、系统写入二次确认；bundle 第三方 license 再分发合规 + SBOM/CVE 门禁。
- 数据隐私：诊断包**不出域**、客户自留，一行写清采集项。
- 可观测：装后自检=离线唯一可观测抓手；失败诊断包可回传。

## 依赖与前置条件
- 完全自包含 → 前置极少（目标机在支持矩阵内即可）；不可 bundle 的依赖列"客户自带"。
- 外部：目标 OS/Python/glibc 在矩阵内。

## 影响评估 · 兼容 · 发布策略
- 纯新增分发路径，不动 SDK 接口/API 预算。
- 安装态向后兼容：离线↔联网命令/自检/（分期）卸载语义对齐；同源产物防漂移。
- 一包=一个冻结 SDK 版本；升级=换整包（原地增量分期）。
- 发布：随 SDK 版本产出签名离线包 + 校验清单。
